# This is what Vibe Coding does to you

![vpc-diagram](https://raw.githubusercontent.com/isudiptodas/isudiptodas_portfolio/main/src/articles/images/vibe-coding-meme.jpg)

If you are someone who ask Claude code, Github Copilot, Cursor to fix this line, fix this error, implement this feature or build this website for you, and after this if you think you are a developer or you know about systems... then I must say you may burst out of anger because in this article you will get very much offended.

**Vibe coder**

- Someone who is too lazy to understand stuff
- Someone who thinks AI is never wrong just like his ex girlfriend
- Someone who thinks typing prompts make them a developer

Now ask a very small question to yourself. That are you actually getting the satisfaction of learning a new thing ? Or you are just being happy that, yeah, you have asked an AI model to do something and it has done that and you are just experiencing or getting the feeling that yeah, you have accomplished something. Is that what is happening with you right now ?

If yes, then I must say you need to stop doing that right now or else in future you will have no option left but to regret.

## Before vibe coding
In earlier days when vibe coding is not even a thing or a fancy term, that time people used to learn stuff either by long tutorials or manually implement stuff. They break things, they learn new things or by reading the documentation or articles. This is how they choose to learn and build stuff and to be honest, this is how real engineers are made.

Previously whenever people have well been assigned with a task or any work or even if they work on a side project, they always had this feeling that if I get stuck anywhere I need to take help of YouTube videos or read documentations and I have to figure this on my own because no one is going to help me out from here. And that was the time when people really used to think. They used to struggle and that is how they made the most use of their memory. To actually learn, implement in their own way and getting the most out of it.

No doubt that after the release of ChatGPT and many other AI tools, people took help from them and even at some point they use these AI tools to automate a lot of processes and reduce the boilerplate code. But that doesn't make them a vibe coder.

## The vibe coding trend
Now in this vibe coding era, what is the actual problem that is visible that people stopped learning new things and implement them in your own. It feels like the creativity is almost dead and it is getting worse day by day.

Either you want to make a new website, OK, ask GitHub Copilot, or you want to write a code for a particular task, OK, ask ChatGPT. Or you need to fix some issues in a codebase, simply open cursor and the entire folder, just import it and start working. Ask cursor to do this and that. He will do that. You will just sit back and relax and watch him do the task. This is the actual reality of today.

Now the actual problem is not that people are using these AI tools. No doubt AI tools leverage human capabilities a lot more effectively. Even personally I also use a lot of AI tools in my day-to-day life. But the problem is when people start to depend on these AI tools like they are literally thinking If I have a cursor or GitHub Copilot, I have access to AI, then I can do anything. I can make any website, any system, any application in minutes. But in reality, things don't work like that. You need to consider and keep a lot of things in mind while creating a system or a website or any particular thing in tech.

***
## Scenario
Suppose you ask Claude code to make a website for a particular business. You prompted their business model, color theme, page style, contact details etc.

But what about :
1.  For frontend you will use vanillaJS or any framework
2. Which backend technology you will prefer and why
3. What about input sanitization & validation
4. How the data will be stored in a database
5. Either you will use a SQL or NoSQL database and why
6. How you will make sure your backend can handle million request without downtime
7. How will you make sure your database health remains stable
8. Your page should load faster
9. How will you handle edge cases and unexpected errors
10. How will you scale the system in tomorrow your user base grows from 1 million to 10 million 

## The effect 
A very major effect of vibe coding is whenever you will encounter a problem in real life in that case, you will first ask an AI model, not just think that what has occurred, the problem or how should I solve it, how should I approach the problem? No, your first response will be I will ask AI. And here in most of the cases, AI don't have large context. So in such cases it may fail to provide you a specific answer and you will get stuck.

Think you are working on a project and that code base grows to around hundreds of files, then folders, subfolders and every file is not getting used. Only some files are getting used and some files are just helper functions which are not needed but when you are stuck on something that suppose the server is not working or a function should return a string but it is returning undefined so how and why it happens ? So in that case you will ask AI. Now AI models have a limited context size or context window. They can remember memory or conversation up to a certain limit after that they will just start to hallucinate. So in that cases, how will you overcome that problem ?

In earlier days, whenever developers encounter an issue, for example, a function is returning a wrong value, a server is crashing every time database is not connecting, frontend can't send request to the backend. Nowadays these issues are considered as normal but back in previous days these are major issues and developers in earlier days used to solve it like this. First they tried to identify that which area is causing the problem. Second they used to approach the problem in a very simple and step by step manner. Then it comes to checking the codebase, checking the logs, checking for any network connections.

And in today's time when people stuck in any place, any error, the first thing they do is not just solve the problem by looking at it, but go to the AI, type a prompt and request it to fix it and also telling 1000 times that please do it correctly without making any mistake. Really fun, right?

## Privacy
Even though in some new age AI tools there is a feature where you have to give them access to the internal file system of your machine and based on that they can update or modify any file, create any file or get access to any file and give answers based on that. Now here arises another question, what about privacy ? Will you intentionally give access to a external AI tool of your machine. Here most of the people will agree to yes why not ? But only a real real tech person who knows that how internally AI processes things, what the AI is capable of or in tech how things actually work, in that case he/she will never agree on this. Privacy is a very high level concern and you can't just rely it on any AI model. No matter how big the company is, how privacy they ensures. Still it is a big issue.

## Conclusion
Now a new rule has been made that if the code given by the AI is working and running perfectly, then everything is fine. Deploy in production or you can use it anywhere. This is the actual problem. In the end the only thing that matters is you can use as much as you want, but don't depend on AI. You still need to understand how the Internet works, how actual system works behind the scenes, because AI can give you code, but AI cannot architect the systems or predict future possibilities and outcomes and edge cases, for that only a human mind will be responsible.
